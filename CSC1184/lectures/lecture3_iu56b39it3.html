Week 3 – NumPy Arrays & Vectorised Operations

In Week 1 we treated lists as tiny datasets and practised loops and summaries. In Week 2 we moved data into files and parsed CSVs into a list of dictionaries. Week 3 introduces NumPy arrays: a more powerful way to store and manipulate numeric data, especially when you have many values.

By the end of this lecture, you should be able to:
  • Explain how NumPy arrays differ from Python lists.
  • Create one-dimensional arrays from Python lists and simple constructors.
  • Use indexing, slicing, and boolean masks to select array elements.
  • Apply vectorised operations and basic aggregations such as mean, sum, min, max, and standard deviation.
  • Take a numeric column from your Week 2 list-of-dicts and move it into a NumPy array for analysis.
Big idea For numeric data, NumPy lets you express “do this to every value” with one operation, instead of writing your own loops again and again.

1. Why NumPy? Lists vs Arrays

Python lists are very flexible: they can store numbers, strings, and even other lists in one container. That flexibility is great when you are getting started, but it has two downsides for data analysis:

  • You often want many numbers of the same type, not a mix of anything.
  • Writing loops over large lists can be slow and verbose.

A NumPy array is designed specifically for numeric work:

  • All elements are usually the same type (for example, all floats).
  • Arrays live in a single block of memory, which makes operations fast.
  • You can apply operations to the whole array at once (vectorised operations).

In this lecture, we keep arrays one-dimensional and think of them as numeric columns: for example, all exam scores or all temperatures.

2. Creating Arrays

Most of the time you will create arrays from data you already have: lists of numbers or numeric columns extracted from CSV files. NumPy also provides convenient functions to create arrays full of zeros, ones, or simple ranges of numbers.

What does import do?

Python is small by default. Many useful tools live in modules (libraries) that you bring into your program with the import statement:

  • import math – makes the math module available (e.g. math.sqrt(9)).
  • import random – gives you random-number functions.
  • import numpy – gives you NumPy’s array tools.

When you import X, you are telling Python: “please load the code for module X so I can call its functions.” You then access those functions with the module name as a prefix, e.g. math.sqrt or numpy.array.

NumPy import & alias reminder

In this module we almost always start our NumPy code with: 

import numpy as np
  • Why as np? numpy is long to type; np is the standard short name in data analysis code.
  • After this, you call functions with the prefix: np.array(...), np.zeros(...), np.mean(...), and so on.
  • You may see other styles (like from numpy import array) online, but for this module we stick to the import numpy as np pattern so code is consistent and easy to read.
Common ways to create a 1D array
  • From a Python list of numbers (most important for this module).
  • From constructors like “all zeros”, “all ones”, or “a range of values”.

4. Indexing, Slicing & Boolean Masks


        

          Indexing and slicing work very similarly to lists: you can access individual elements
          or ranges by position. The new ingredient for arrays is the boolean mask:
          an array of True/False values that lets you filter based on a condition.
        


        
4.1 Indexing and slicing (review)


        
    
          
  • scores[0] – first element.
    • 
          
  • scores[1:4] – elements at positions 1, 2, 3.
    • 
          
  • scores[-1] – last element.
    • 
        


        
4.2 Boolean masks – asking questions as filters


        

          If you compare an array to a value, you get another array of booleans:
          one True or False per element. You can then use that boolean
          array to pick out just the elements you care about.
        


        

          Mask pattern
          
    
            
  1. Start with an array of numbers, e.g. exam scores.
    2. 
            
  2. Build a condition like “score ≥ 60” → this creates a boolean mask.
    3. 
            
  3. Apply the mask to the array to keep only the True positions.

5. Vectorised Operations & Summaries


        

          A major benefit of arrays is that operations are defined “elementwise”:
          if you add 5 to an array, you get a new array where each element increased by 5. You
          do not need to write the loops yourself.
        


        
5.1 Vectorised arithmetic


        
    
          
  • scores + 5 – add 5 to every score.
    • 
          
  • scores / 100 – convert scores to proportions between 0 and 1.
    • 
          
  • scores * 2 – double all scores (not always meaningful, but the pattern holds).
    • 
        


        
5.2 Aggregations: mean, sum, min, max, standard deviation


        

          Arrays provide methods to compute common summaries directly:
          scores.mean(), scores.sum(), scores.min(),
          scores.max(), scores.std(), and so on. These are conceptually
          the same as the built-ins and formulas you used before; they are just packaged for you.
