Update load_data so that it can handle the messy file without raising errors.

          
    
            
  • Skip completely empty lines (you can use if not line.strip(): continue).
    •

    • 
            
  • 
              When converting to int, only do so if the value looks numeric, e.g.
              if text.isdigit(): value = int(text). Otherwise, leave it as a string.

7. Final Challenge: Reusability

        

          You also downloaded sales_data.csv. This file has completely different columns
          (e.g. product, price, quantity) compared to the student files.
        


        

          

            
The Problem

            

              If your loader contains hard-coded column names (like row["exam_score"]),
              it will break when used with a different dataset.
            

            

              For a generic loader, keep it responsible only for:
              "turn lines into labelled dictionaries".
            

          

          

            
The Challenge

            

              Make your load_data function generic:
            

            
    
              
  • Do not hard-code column names into the loader.
    • 
              
  • Return strings for each field.
    • 
              
  • Do dataset-specific conversions (e.g. string → number) in your analysis code.
