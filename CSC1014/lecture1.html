Week 1 – What Is Programming?

Over the next two hours, you will:

  • Describe what a computer does at a high level.
  • Explain what a program is and why it must be precise.
  • Recognise everyday activities as algorithms.
  • Understand how algorithms are translated into programming languages.
  • Briefly meet Python, the language we’ll use in this module.
Big Picture Today is about concepts, not details. If you leave this lecture seeing recipes, instructions, and checklists as algorithms, you’re in exactly the right place.

1. What Is a Computer?

A computer is an electronic device that can store data and follow instructions to transform that data. Although modern computers look very different (phones, laptops, watches, car dashboards), a lot of them share the same basic structure.

At a high level, we can think of a computer as having:

  • Input devices – keyboard, mouse, touchscreen, microphone, camera.
  • Output devices – screen, speakers, printer, vibration motor.
  • Memory – where data and instructions are stored.
  • CPU (Central Processing Unit) – the “brain” of the computer:
    • Performs calculations and comparisons.
    • Decides which instruction to run next.

The CPU repeatedly performs a simple cycle:

  1. Fetch the next instruction from memory.
  2. Decode it (figure out what it means).
  3. Execute it (do the operation: add, compare, jump, etc.).
Key Idea A computer is fast but stupid. It can perform billions of simple operations per second, but only if we provide precise, unambiguous instructions.

Mini activity (2–3 minutes):

  • List three devices you used today that count as computers (not just “laptop”).
  • For each one, identify one input and one output.
  • Notice how many computers are “hidden” in everyday objects.

2. What Is a Program?

A program is a precise sequence of instructions that a computer can execute. It tells the computer:

  • What data to use (input).
  • What steps to perform (processing).
  • What results to produce (output).

2.1 Human Language vs Computer Language

Humans are good at dealing with vague instructions: “Can you send me that thing later?” usually works. Computers are terrible at this. They require instructions that are:

  • Exact – no missing steps.
  • Unambiguous – only one possible meaning.
  • Complete – everything needed to reach the goal is specified.
Important A computer cannot guess what you meant. If the instructions are vague, contradictory, or incomplete, the program will either crash or do the wrong thing.

2.2 Data and Programs in Memory

Modern computers store programs in memory alongside data. The same memory chip might hold:

  • The instructions that define your web browser.
  • The text of the page you’re reading.
  • Your typed responses and settings.

This is called the stored-program concept. You don’t need to remember the name, but it explains why computers can load and run different programs so flexibly.

3. Algorithms: Step-by-Step Instructions

Before we write code, we need to describe how to solve a problem. An algorithm is:

A step-by-step method for solving a problem or achieving a goal.

You already use algorithms in everyday life: recipes, directions, assembly instructions, checklists, and so on.

3.1 Properties of a Good Algorithm

  • Input – what information does it start with?
  • Output – what result does it produce?
  • Definiteness – each step is clear and unambiguous.
  • Finiteness – it eventually finishes, rather than looping forever.
  • Effectiveness – each step is simple enough to be carried out mechanically.

3.2 Everyday Example: Making a Cup of Tea

Informal version:

  1. Boil water.
  2. Put a teabag in a cup.
  3. Pour boiling water into the cup.
  4. Wait a few minutes.
  5. Remove the teabag.
  6. Add milk/sugar if desired.
  7. Drink.

For a computer (or a very literal robot), we would need to be much more precise: how much water? how hot? how long is “a few minutes”? what if there is no teabag?

Programming Mindset When you think like a programmer, you start noticing all the hidden assumptions in everyday instructions. Programming is partly about removing those assumptions.

3.3 Activity: Write an Everyday Algorithm

(10–15 minutes, discussion in class or self-reflection)

Pick one of these tasks (or invent your own):

  • Brushing your teeth.
  • Locking up your house when you leave.
  • Making a sandwich.

Write down the steps as if explaining to a very literal robot:

  • Put one action per line.
  • Avoid vague words like “some”, “a bit”, “properly”.
  • Look for decision points: If there is no bread, then…
  • Look for repetition: Stir until no lumps remain.

4. From Algorithms to Code

A computer cannot understand human languages directly. We need a way to turn our algorithm (idea) into something a computer can execute. This is where programming languages come in.

4.1 Levels of Description

  1. Informal description – “Find the largest of three numbers.”
  2. Algorithm (natural language or pseudocode) – step-by-step instructions that a human can follow.
  3. Code – the algorithm written in a formal programming language like Python.

4.2 Pseudocode (Not Real Code Yet!)

Pseudocode looks a bit like programming, but it’s written for humans, not machines. There is no single “correct” syntax; the goal is clarity.

Example: find the largest of three numbers a, b, c:

Set largest to a
If b is greater than largest:
    Set largest to b
If c is greater than largest:
    Set largest to c
Output largest

Different people might write this slightly differently, and that’s fine. Once the algorithm is clear, we can translate it into Python in a later lecture.

4.3 Activity: Pseudocode for a Simple Problem

(10 minutes)

Write pseudocode for one of these tasks:

  • Given three test scores, compute the average and say whether it is a pass or fail.
  • Given a person’s age, decide whether they are a child, teenager, adult, or senior.

Focus on the logic, not on Python syntax. Ask yourself:

  • What are the inputs?
  • What is the output?
  • What decisions (if any) are needed?

5. Meet Python

In this module we will use the Python programming language. Python is widely used in web development, data science, artificial intelligence, automation, and more.

  • Its syntax is relatively simple and readable.
  • It runs on many platforms (Windows, macOS, Linux, the web).
  • There is a huge ecosystem of libraries and learning resources.
