Week 4 – Input, Output & Simple Interactive Programs

In Weeks 2–3 you used Python like a calculator and learned how to store results in variables. This week you connect your programs to the outside world: asking the user for information with input() and presenting results clearly with print() and simple printf-style formatting.

Lecture goals
  • Understand why interactive programs need user input.
  • Use print() precisely, including multiple arguments and basic formatting ideas.
  • Use input() to read data as text from the user.
  • Convert string input into numbers using int() and float().
  • Combine input, variables, and expressions into small interactive scripts.
  • See how comparisons can be applied to user-provided data (preview for if).
  • Recognise and interpret common errors related to input and conversion.

1. Why Do We Need Input?

So far, every number in your programs was typed directly into the code: (12 + 15 + 19) / 3, 8 * 15, 125 // 30, and so on. This is sometimes called hard-coded data. The program always does the same calculation with the same values unless you edit the code.

That is fine for quick experiments, but most useful programs need to work with different data each time they run:

  • A currency converter should work for any amount the user types.
  • A grade calculator should work for any student’s marks.
  • A time estimator should work for any distance and speed.

Instead of changing the code for each scenario, we want:

Ask
→
Read
→
Store
→
Compute
→
Show result

The functions print() and input() work together to support this pattern:

  • print() – sends information out to the user.
  • input() – brings information in from the user.
Activity – Programs that need input

Think of three tiny programs that clearly need user input, for example:

  • “Ask the user for a temperature in Celsius and print it in Fahrenheit.”
  • “Ask for a distance in kilometres and print it in miles.”
  • “Ask for a number of days and print the equivalent in hours.”

We will essentially build variations of these today.

2. The print() Function (and printf)

You have already used print() in earlier weeks, but we mostly treated it as “magic that makes text appear”. Now we look more carefully at what it does and how it relates to the common idea of printf-style formatting.

2.1 What print() actually does

The print() function:

  • Takes zero or more arguments (values you give it).
  • Converts each argument to a string representation.
  • Joins them with a separator (by default, a single space).
  • Sends the resulting line of text to the output, followed by a newline.

3. The input() Function


        

          Now we introduce the new piece: input(). This function lets your program
          pause and wait for the user to type something.
        


        
3.1 Basic behaviour


        

          The pattern is:
        


        
variable_name = input(prompt_string)
        


        

          When Python reaches this line:
        

        
    
          
  1. It prints the prompt_string (without a newline at the end).
    2. 
          
  2. It waits for the user to type a line of text and press Enter.
    3. 
          
  3. It takes everything the user typed (except the Enter) as a string.
    4. 
          
  4. It stores that string in variable_name.
    5.

4. Converting Input to Numbers


        

          If you ask the user for their age or a distance, you probably want to use that value
          in a numeric expression (add, multiply, compare). But input() gives you a
          str, not an int or float.
        


        

          To use the value in calculations, you need to convert the string to a number:
        


        
    
          
  • int("21") → 21 (integer)
    • 
          
  • float("3.5") → 3.5 (floating-point number)
    • 
        


        
4.1 Two-step pattern: safer for beginners


        

          We will start with a two-step pattern:
        


        
age_text = input("Enter your age: ")   # this is a string
age = int(age_text)                    # convert string to int

          Use int() when you expect a whole number (e.g. age in years, number of items).
          Use float() when you expect a decimal number (e.g. hourly pay, temperature).

7. Errors with Input & Conversion


        

          As soon as your program starts reading user input, there are more ways things can go wrong.
          It is important to be able to recognise the main error types and roughly what they mean.
        


        
7.1 Quick review of error types


        
    
          
  • 
            SyntaxError – the code is not valid Python.
            
    Example: forgetting a closing parenthesis:

    print("Hello"   # SyntaxError

    
          
    • 
          
  • 
            TypeError – the operation makes no sense for the types involved.
            
    Example from last week:

    "2" - "1"       # TypeError

    
          
    • 
          
  • 
            ValueError – the value is the wrong shape/content for the operation,
            even though the type is string.
            
    Example:

    int("hello")    # ValueError
int("3.5")      # ValueError (but float("3.5") is okay)

    
          
    • 
        


        
7.2 ValueError with input conversion


        

          When you convert user input with int() or float(), a
          ValueError can occur if the user’s text is not a valid number.

8. Recap & Self-check


        
8.1 Key ideas from Week 4

        
    
          
  • print() displays values; it can take multiple arguments separated by commas.
    • 
          
  • Formatted output (printf-style) means writing a template string and filling in values;
              in Python, f-strings are a convenient way to do this.
    • 
          
  • input() pauses the program, shows a prompt, and returns a str
              containing what the user typed.
    • 
          
  • To use input in arithmetic, you must convert it with int() or
              float().
    • 
          
  • Variables store user-provided values just like hard-coded ones.
    • 
          
  • Comparisons can be applied to input data to compute Booleans (preview of using if).
    • 
          
  • Common errors include SyntaxError, TypeError, and ValueError when converting invalid input.
    •
