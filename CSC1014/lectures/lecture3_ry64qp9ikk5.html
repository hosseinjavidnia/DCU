Week 3 – Variables & State: Remembering Data

In Week 2, Python behaved like a calculator: you typed expressions, and it produced answers. Today we move to real programs that can remember information as they run, by using variables.

1. Why Do We Need Variables?

In the previous lecture, you wrote expressions like (12 + 15 + 19) / 3 or 20 // 6. Python printed the answers, but then the answers disappeared. If you wanted to use the same result again, you had to re-type the whole expression.

Imagine you want to do this:

  • Compute a total price.
  • Compute tax based on that price.
  • Print both the price before tax and the price after tax.

Without variables, you end up writing long, repetitive expressions everywhere, and if you change the underlying numbers, you must update them in multiple places. This is error-prone and hard to read.

Key idea A variable is a name that refers to a value stored in memory. It lets a program remember data and re-use it in later calculations.

Once we introduce variables, we can say “price” instead of repeatedly writing the full expression that produced the price. Variables make programs:

  • Shorter and less repetitive.
  • Easier to understand when you read them.
  • Easier to change later (“What if the tax rate changes?”).
Activity – Where would you use variables?

Think of one everyday situation where you calculate something and then use that result again later – for example:

  • Working out total study hours in a week.
  • Calculating a student’s average grade, then checking if they passed.
  • Estimating a budget and then adding different expenses to it.

In each case, what would you want to “remember” as you work through the steps?

2. Memory & Named Boxes

Inside your computer there is memory that can store many values at once. The machine doesn’t think in terms of “variables”; it just has cells of memory that contain bits. Programming languages like Python give us a friendlier view: we can imagine each value living inside a box with a label on it.

2.1 The “named box” model

Picture this situation in a program:

  • You have a box labelled age that contains the value 21.
  • You have a box labelled name that contains the text "Ada".
  • You have a box labelled is_student that contains True.

In your head, you might imagine something like: 

+-----------+---------+
| name      |  Ada    |
+-----------+---------+

+-----------+---------+
| age       |   21    |
+-----------+---------+

+-----------+---------+
| is_student|  True   |
+-----------+---------+

The important part of this picture is the separation between the label (the variable name) and the content (the value). The content can change over time, but the label still points to “the same box”.

2.2 Updating the contents

Suppose you have a box called count that currently contains 0. If your program processes another item, you might want to increase count by 1. After the update, the same box count now contains 1.

Over time, you can imagine memory evolving as the program runs: 

Initially:
+--------+-----+
| count  |  0  |
+--------+-----+

After one update:
+--------+-----+
| count  |  1  |
+--------+-----+

After two updates:
+--------+-----+
| count  |  2  |
+--------+-----+

This idea of memory changing as the program runs is called the program’s state. Variables are how we observe and control that state.

3. Assignment in Python

Now we connect the mental model to Python syntax. In Python, we create or update a variable using an assignment statement.

3.1 General form

The general pattern of assignment is: 

name = expression

When Python executes an assignment, it follows this order:

  1. Evaluate the expression on the right-hand side (RHS).
  2. Store the result in the variable named on the left-hand side (LHS).

If the variable name is being used for the first time, Python will create a new box. If it already exists, Python will replace the old value in that box with the new one.

3.2 Assignment vs equality

        

          In mathematics, an equation like x = x + 1 makes no sense:
          one side cannot be both equal to and bigger than the other side at the same time.
        


        

          In programming, x = x + 1 has a different meaning:
        

        
    
          
  • Compute the value of x + 1 using the old value of x.
    • 
          
  • Store that new value back into the box labelled x.
    • 
        


        

          So if x was 4, after x = x + 1, x becomes 5.

4. Choosing Good Variable Names


        

          Variables are for humans as much as they are for the computer. The computer does not care
          whether you call a variable x or total_price_with_tax, but a human
          reader will find the second one much easier to understand.
        


        
4.1 Rules for names

        

          In Python, a variable name:
        

        
    
          
  • Can use letters (a–z, A–Z), digits (0–9), and underscores (_).
    • 
          
  • Cannot start with a digit.
    • 
          
  • Is case-sensitive: total, Total, and TOTAL are different.
    • 
          
  • Cannot be a reserved keyword like if, for, True, etc.
    • 
        


        
Legal names (syntactically correct):

        
    
          
  • age, total_price, student_count
    • 
          
  • _private_value, number2
    • 
        


        
Illegal names (will cause errors):

        
    
          
  • 2nd_place (starts with a digit)
    • 
          
  • my-name (contains a hyphen)
    • 
          
  • class, if, for (keywords)
    • 
        


        
4.2 Style: making code readable

        

          Beyond the strict rules, there are style conventions that make code more readable:
        

        
    
          
  • Use lowercase_with_underscores (called “snake_case”).
    • 
          
  • Choose descriptive names: hours_worked, hourly_rate, pay.
    • 
          
  • Avoid one-letter names, except for very short-lived values in simple examples.
    •

6. Variables and Types


        

          A variable doesn’t just store “something”; it stores a value of a particular
          type. The type comes from the value currently inside the box.
        


        

          For example:
        

        
    
          
  • age = 21 → age holds an int.
    • 
          
  • pi = 3.14159 → pi holds a float.
    • 
          
  • name = "Ada" → name holds a str.
    • 
          
  • is_student = True → is_student holds a bool.

7. Preview: Counters & Accumulators


        

          Many programs use the same update patterns over and over. Two of the most important are:
        


        
    
          
  • 
            Counter: a variable that increases by 1 each time something happens.
            
    
            Example idea: counting how many numbers have been processed.
          
    • 
          
  • 
            Accumulator: a variable that keeps a running total.
            
    
            Example idea: summing up all exam scores.
          
    • 
        


        

          In code, these patterns look like:
        


        
# Counter pattern (concept)
count = 0      # start from zero
count = count + 1   # one more item
count = count + 1   # one more item
        


        
# Accumulator pattern (concept)
total = 0
total = total + 5
total = total + 10
