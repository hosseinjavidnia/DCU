Lecture goals

Big picture So far, your programs could only do things once and make one-off decisions. Today you give them the power to repeat actions automatically and stop when a condition is met.
Key terms (used in this lecture)
  • Loop: a structure that repeats a block of code while a condition is true.
  • Iteration: one complete run of the loop body from top to bottom.
  • Loop condition: the Boolean expression that decides whether the loop runs again.
  • Infinite loop: a loop whose condition never becomes false, so it never stops.
  • Counter-controlled loop: a loop where a counter variable controls how many iterations occur.
  • Accumulator loop: a loop that builds up a running total in a variable.
  • Off-by-one error: a logic mistake where a loop runs one time too few or too many.
  • Trace table: a table used to follow variable values line by line through a program.

1. Why Do We Need Loops?

In everyday life, many tasks involve repetition. For example:

  • Climbing stairs: repeat “step up” until you reach the top floor.
  • Brushing teeth: repeat the brushing motion until 2 minutes have passed.
  • Checking email: repeat “check inbox” every morning.

In programming, we often need to repeat operations such as:

  • Adding up a list of numbers.
  • Asking the user again if they typed invalid input.
  • Simulating several time steps in a simple model.

Without loops, the only way to repeat something is to write the same code line many times. This is:

  • Hard to read.
  • Hard to change (if you want to go from 10 repetitions to 100).
  • Easy to get wrong.
Key idea A loop lets you say “repeat this block of code while this condition is true” instead of copy-pasting the block many times.

2. What Is a Loop?

At a high level, a loop is a control structure that repeats a block of code. Each time the block runs is called an iteration.

Every loop involves three main parts:

  1. Initialisation – set up any variables used to control the loop.
  2. Condition – a Boolean expression checked before each iteration.
  3. Update – code inside the loop body that changes something so the condition will eventually become false.

In Python, we will use the while loop to implement this pattern.

3. The while Loop: Syntax & Meaning

Python’s while loop has this general form: 

while <condition>:
    <indented block of one or more statements>

The condition must be a Boolean expression: something that evaluates to either True or False. For example:

  • x <= 10
  • count != 0
  • not done

Python executes a while loop like this:

  1. Evaluate the condition.
  2. If the condition is True:
    • Execute the indented block (one iteration).
    • Go back to step 1.
  3. If the condition is False, skip the block and continue after the loop.

5. Classic while Loop Patterns

        

          Many loops in real programs follow a small number of common patterns. Recognising these
          patterns makes code easier to understand and write.
        


        
5.1 Counter-controlled loop

        

          A counter-controlled loop uses a variable (the counter) to count
          how many iterations have happened. The loop condition typically compares the counter to
          a limit.
        


        

          Template: Counter-controlled loop

counter = start_value        # initialisation
 while counter <= end_value:  # condition
    # loop body using counter
    counter = counter + 1    # update

        


        
Typical uses:

        
    
          
  • Printing numbers from 1 to 10.
    • 
          
  • Repeating a simple action a fixed number of times.
    • 
        


        
5.2 Accumulator loop

        

          An accumulator loop keeps a running total in a variable, often named
          total, sum, or count. Each iteration adds something
          to the accumulator.
        


        

          Template: Accumulator loop

total = 0                   # accumulator initial value
 counter = 1                 # or some start value
 while counter <= n:
     total = total + counter # update accumulator
     counter = counter + 1   # update counter

 print(total)

        


        
Typical uses:

        
    
          
  • Adding up a sequence of numbers.
    • 
          
  • Counting how many values satisfy a condition.
    • 
        


        
5.3 Sentinel loop (preview)

        

          A sentinel loop reads values until a special “sentinel” value appears.
          A sentinel value is a special value used only to signal “stop”, for example
          0 or -1.
        

        

          Example idea: keep asking for numbers until the user types 0, then stop and
          print the sum. We will use this pattern more in labs and later lectures.

7.2 Off-by-one errors

        

          An off-by-one error happens when your loop runs one time too many
          or one time too few. This often comes from using < instead of
          <=, or starting at 0 when you meant to start at 1.
        

        

          Example (conceptual): the intention is to print numbers 1 to 5, but the program prints
          only 1 to 4, or 0 to 5. Trace tables help catch these mistakes.
        


        
        
8. How Many Times Does the Loop Run?

        

          When you see a loop, it is useful to ask:
          “How many iterations will this take?”
        

        

          For a loop like:
        


        
i = 1
 while i <= n:
     # body
     i = i + 1


        

          the number of iterations grows roughly “with n”. If n doubles,
          the number of iterations also doubles. This is an informal view of
          time complexity – how the running time of a program grows as the size of
          its input grows.
        


        

          Informal complexity
          We will not use formal complexity notation in this module, but you should notice
          that a loop doing 10 steps is “lighter” than a loop doing 1,000,000 steps.
