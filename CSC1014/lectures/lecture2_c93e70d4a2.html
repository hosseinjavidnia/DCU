1. Data, Values & Types

When we write programs, we are always working with data. Some examples:

  • A person’s age: 21
  • A bank balance: 1234.56
  • A name: "Ada"
  • A yes/no answer: True (for “yes”), False (for “no”)

1.1 Values

A value is a specific piece of data your program can work with: 7, 3.14, "hello", True, and so on.

Examples of values

0, -5, 100

2.5, 0.0, -3.14

"cat", "123", "Hello, world!"

True, False

1.2 Types

Every value in Python has a type. The type tells Python:

  • What kind of data this is (number? text? True/False?).
  • What operations make sense on it (add? compare? join?).
  • How to store it in memory.
Mental model Think of a type as a label on a storage box:
  • A box labelled “integer” can hold whole numbers.
  • A box labelled “string” can hold text.
  • A box labelled “bool” can only hold True or False.
The label helps the computer know how to treat the value inside.

1.3 Why different types?

Why not just store everything as “stuff”? Because different operations make sense for different kinds of data:

  • For ages and bank balances, we want to add, subtract, compare sizes.
  • For names, we want to join pieces, change case, search for substrings.
  • For yes/no answers, we want to make decisions: “if this is True, do X.”
Common confusion On the screen, "7" and 7 look similar, but:
  • 7 is a number.
  • "7" is text that happens to contain the character 7.
They have different types, and Python treats them differently.

1.4 A very rough memory picture

Inside the computer, values are stored as patterns of bits (0s and 1s). The type tells Python how to interpret those bits:

  • As a whole number (integer)
  • As a number with a decimal point (floating-point)
  • As characters of text (string)
  • As a truth value (Boolean)

2. Basic Types in Python

Python has many types, but in this module we’ll mostly use four basic ones:

Integers – int

Whole numbers: 0, 7, -42, 2025.

Floating-point numbers – float

Numbers with a decimal part: 3.14, -0.5, 2.0.

Booleans – bool

Truth values: True or False.

Strings – str

Text: "hello", "CS1014", "123".

Python can tell you the type of a value using the function type().

3. Expressions & Order of Operations


        

          A program is made of statements (full instructions) and
          expressions (pieces that produce values).
        

        

          An expression is something that Python can evaluate to a single value:
        

        
    
          
  • 2 + 3
    • 
          
  • 10 - 4
    • 
          
  • 3 * (2 + 1)
    • 
          
  • 7 > 3
    • 
          
  • "Hello " + "world"
    • 
        


        
3.1 Operators

        
For numbers, we’ll use these basic operators:

        
    
          
  • + — addition
    • 
          
  • - — subtraction
    • 
          
  • * — multiplication
    • 
          
  • / — (true) division – result is a float
    • 
          
  • // — integer division — drops the decimal part
    • 
          
  • % — remainder (modulo)
    • 
          
  • ** — exponent (power)
    • 
        


        
3.2 Order of operations (precedence)

        

          When an expression has more than one operator, Python follows rules called
          precedence, similar to maths:
        

        
    
          
  • Parentheses first: ( ... )
    • 
          
  • Then powers: **
    • 
          
  • Then multiplication, division, integer division, modulo: * / // %
    • 
          
  • Then addition and subtraction: + -
    • 
          
  • Left-to-right when operators have the same precedence.
    •

7. Recap & Self-check


        
7.1 Key ideas from Week 2

        
    
          
  • A value is a specific piece of data; every value has a type.
    • 
          
  • Python’s basic types in this module: int, float, bool, str.
    • 
          
  • type() shows the type of a value or expression.
    • 
          
  • An expression combines values and operators to produce a new value.
    • 
          
  • Operator precedence determines the order in which parts of an expression are evaluated; parentheses can override it.
    • 
          
  • Comparison expressions produce Boolean values (True/False) of type bool.
    • 
          
  • Strings represent text and behave differently from numbers, even if they “look numeric”.
