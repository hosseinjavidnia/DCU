Week 5 – Making Decisions with if Statements

So far your programs always followed the same sequence of steps: read input, do calculations, print results. In this lecture you will learn how to make your programs choose between different actions using Boolean conditions and if statements.

Lecture goals
  • Understand why programs need decision-making, not just calculation.
  • Use Boolean expressions as conditions that are either True or False.
  • Write one-way if statements that conditionally execute a block of code.
  • Write two-way decisions using if / else.
  • Write multi-way decisions using if / elif / else.
  • Recognise and avoid common mistakes with if statements (indentation, = vs ==, unreachable branches).
Where are we?
  • Week 3: variables and state – programs can remember values.
  • Week 4: input and output – programs can talk to users and use their data.
  • Week 5: decisions – programs can choose actions based on that data.

1. Why Programs Need Decisions

Think about how often you make decisions in everyday life:

  • If it is raining, take an umbrella; otherwise, wear sunglasses.
  • If you pass the exam, you get credit; otherwise, you repeat the module.
  • If your bank balance is negative, apply an overdraft fee.

Our programs should be able to react in similar ways:

  • If the user is under 18, show a different message.
  • If the score is above a pass mark, print “Pass”; otherwise, “Fail”.
  • If the input is negative, show a warning; otherwise, continue normally.

Without decision-making, a program can only do the same thing every time, regardless of the data. It can calculate, but it cannot choose.

Key idea Decision-making in programs is about asking a yes/no question and then choosing between different blocks of code depending on the answer.

2. Boolean Expressions as Conditions

In Week 2, you saw that Python has Boolean values True and False, and that comparison operators produce these values:

  • age >= 18
  • score == 100
  • balance < 0

These are Boolean expressions. They are the “questions” your program can ask: “Is this condition true?” A Boolean expression is a good condition if it can only be True or False.

2.1 Comparisons and logical operators

Quick reminder:

  • Comparison operators: <, >, <=, >=, ==, !=
  • Logical operators: and, or, not

We can combine these to form richer conditions:

  • age >= 18 and age <= 65
  • score >= 40 or bonus_points > 0
  • not (status == "cancelled")

3. One-Way if Statements


        

          The simplest decision is a one-way if: do something only if a condition
          is true, otherwise skip it.
        


        
3.1 Structure and indentation


        

          Conceptually, the structure is:
        


        
if <condition>:
    <indented block of one or more statements>

<next statement after the if>
        


        

          Execution steps:
        

        
    
          
  1. Evaluate the condition once.
    2. 
          
  2. If it is True, execute the indented block.
    3. 
          
  3. If it is False, skip the block entirely.
    4. 
          
  4. Then continue with the next unindented statement after the block.
    5. 
        


        

          Indentation (spaces at the start of a line) is how Python knows which statements belong
          to the if. All lines in the block must be indented consistently.

4. Two-Way Decisions with if / else


        

          A one-way if handles “maybe do this extra thing”. Often we want exactly one
          of two alternatives: if the condition is true, do A; otherwise, do B. This is a two-way
          decision.
        


        
4.1 Structure


        
if <condition>:
    <block A>
else:
    <block B>
        


        

          Exactly one of the two blocks will run:
        

        
    
          
  • If the condition is True → run block A, skip block B.
    • 
          
  • If the condition is False → skip block A, run block B.
    •

5. Multi-Way Decisions with if / elif / else


        

          Many real problems have more than two categories: multiple letter grades, several age
          groups, or different ticket prices. Writing separate, disconnected if
          statements can lead to overlapping logic. Instead, we use a single ordered chain:
        


        
if <condition1>:
    <block 1>
elif <condition2>:
    <block 2>
elif <condition3>:
    <block 3>
else:
    <default block>
        


        

          Python evaluates the conditions in order:
        

        
    
          
  1. If condition1 is true, run block 1 and skip all others.
    2. 
          
  2. Otherwise, if condition2 is true, run block 2 and skip the rest.
    3. 
          
  3. Otherwise, if condition3 is true, run block 3 and skip the rest.
    4. 
          
  4. If none of the conditions is true, run the else block (if present).
    5. 
        


        

          At most one block runs.

6. Common Pitfalls with if Statements


        
6.1 Assignment (=) vs comparison (==)


        

          One of the most common beginner mistakes is to confuse assignment (=) with
          comparison (==):
        

        
    
          
  • x = 5 means “store 5 in the variable x”.
    • 
          
  • x == 5 means “check whether x is equal to 5”.
    • 
        


        

          Inside an if condition, you almost always want ==, not =.

6.2 Indentation errors


        

          Python uses indentation to mark blocks. Common mistakes:
        

        
    
          
  • Forgetting to indent a line that should be inside an if block.
    • 
          
  • Accidentally indenting a line that should be outside the block.
    • 
          
  • Mixing tabs and spaces (we recommend 4 spaces per indentation level).
    • 
        


        

          If the indentation is wrong, Python may raise an IndentationError or your
          program may run but behave differently from what you intended.
        


        
6.3 Unreachable or overlapping branches


        

          In an if / elif chain, the order of conditions matters.
          A common logical bug is to write overlapping conditions so that some branches can never
          be taken.
        


        
For example, imagine:


        
if score >= 40:
    print("Pass")
elif score >= 80:
    print("Distinction")
        


        

          The second branch will never run: any score ≥ 80 already satisfies score >= 40,
          so the first branch always catches it. Instead, the more specific condition should appear first.

8. Recap & Self-check


        
8.1 Key ideas from Week 5

        
    
          
  • Programs need decision-making to react differently to different input.
    • 
          
  • Boolean expressions (True/False) are used as conditions in if statements.
    • 
          
  • A one-way if optionally executes a block if the condition is true.
    • 
          
  • if / else chooses exactly one of two blocks.
    • 
          
  • if / elif / else chooses among many alternatives, in order.
    • 
          
  • Assignment (=) and comparison (==) have different meanings.
    • 
          
  • Indentation defines which statements belong to which block.
    • 
          
  • Ordering conditions correctly avoids unreachable branches.
    • 
        


        
8.2 Concept quiz

        

          
    
            
  1. 
              For the code:

    x = 10
if x > 5:
    print("A")
print("B")

    
              What is printed?
              
    
                Show answer
                
    
                  Both A and B. The condition is true, so the if
                  block runs, and then execution continues with the next statement.
                
    
              
    
            
    2. 
            
  2. 
              For the code:

    score = 75
if score < 40:
    grade = "Fail"
elif score < 60:
    grade = "Pass"
elif score < 80:
    grade = "Merit"
else:
    grade = "Distinction"

    
              What is the final value of grade?
              
    
                Show answer
                
    
                  "Merit". The first two conditions are false; the third
                  (score < 80) is true, so that block runs and the rest are skipped.
                
    
              
    
            
    3. 
            
  3. 
              Why is this code logically wrong for distinguishing pass and distinction?

    if score >= 40:
    print("Pass")
elif score >= 80:
    print("Distinction")

    
              
    
                Show answer
                
    
                  Any score ≥ 80 also satisfies score ≥ 40, so the first branch always
                  catches it and the second branch is never reached. The more specific condition
                  should come first.
