Week 1 – What Is Programming?

Welcome to CSC1014. This week we are not trying to turn you into a programmer overnight. Instead, we are building the mental model you will use for the rest of the module: what computers are, what programs are, and how everyday instructions relate to algorithms.

Lecture goals
  • Describe what a computer does at a high level.
  • Explain what a program is and why precision matters.
  • Recognise everyday activities as algorithms.
  • Understand how algorithms are translated into programming languages.
  • Briefly meet Python, the language we’ll use in this module.
Big picture Today is about concepts, not details. If you leave this lecture seeing recipes, instructions, and checklists as algorithms, you’re in the right place.

1. What Is a Computer?

A computer is an electronic device that can store data and follow instructions to transform that data. Although modern computers look very different (phones, laptops, watches, car dashboards), a lot of them share the same basic structure.

At a high level, we can think of a computer as having:

Input devices

Keyboard, mouse, touchscreen, microphone, camera.

Output devices

Screen, speakers, printer, vibration motor.

Memory

Where data and instructions are stored.

CPU

Performs calculations and decides which instruction to run next.

The CPU repeatedly performs a simple cycle:

Fetch
→
Decode
→
Execute
Key Idea A computer is fast but stupid. It can perform billions of simple operations per second, but only if we provide precise, unambiguous instructions.

1.1 Different kinds of computers

When we say “computer”, most people picture a laptop or desktop. But from a programming point of view, many everyday objects count as computers:

  • A smartphone running apps.
  • A smartwatch tracking your steps.
  • A car’s onboard system controlling engine and sensors.
  • A games console, smart TV, or even your washing machine.

All of these contain one or more CPUs, some memory, and programs that control how they behave.

1.2 Myth vs reality

Myth: A computer is “smart”.
Reality: A computer follows instructions blindly and literally.
Myth: Programming means “talking to the computer”.
Reality: Programming means writing precise rules that the computer obeys.

Mini activity (2–3 minutes):

  • List three devices you used today that count as computers (not just “laptop”).
  • For each one, identify one input and one output.
  • Notice how many computers are “hidden” in everyday objects.

2. What Is a Program?

A program is a precise sequence of instructions that a computer can execute. It tells the computer:

  • What data to use (input).
  • What steps to perform (processing).
  • What results to produce (output).

2.1 Programs you already use

Even if you’ve never written code before, you probably use dozens of programs every day:

  • Your web browser, email client, and messaging apps.
  • Maps and navigation apps that compute routes.
  • Photo and video apps that transform images and sound.
  • Banking apps, online shopping sites, games, and more.

Behind each of these is code: someone wrote instructions that describe exactly how the program should respond to your input.

2.2 Human Language vs Computer Language

Humans are good at dealing with vague instructions: “Can you send me that thing later?” usually works. Computers are terrible at this. They require instructions that are:

  • Exact – no missing steps.
  • Unambiguous – only one possible meaning.
  • Complete – everything needed to reach the goal is specified.
Important A computer cannot guess what you meant. If the instructions are vague, contradictory, or incomplete, the program will either crash or do the wrong thing.

2.3 Data and Programs in Memory

Modern computers store programs in memory alongside data. The same memory chip might hold:

  • The instructions that define your web browser.
  • The text of the page you’re reading.
  • Your typed responses and settings.

This is called the stored-program concept. You don’t need to remember the name, but it explains why computers can load and run different programs so flexibly.

Think–pair–share (5 minutes)

(Or simply think + jot down notes if you are working alone.)

  1. Pick one app or website you use a lot.
  2. List three things it can do in response to your actions (e.g. “when I click X, it does Y”).
  3. For each one, imagine the invisible program instruction behind it.

3. Algorithms: Step-by-Step Instructions

Before we write code, we need to describe how to solve a problem. An algorithm is:

A step-by-step method for solving a problem or achieving a goal.

You already use algorithms in everyday life: recipes, directions, assembly instructions, checklists, and so on.

3.1 Properties of a Good Algorithm

Input

What information does it start with?

Output

What result does it produce?

Definiteness

Each step is clear and unambiguous.

Finiteness

It eventually finishes, rather than looping forever.

Effectiveness

Steps are simple enough to be carried out mechanically.

3.2 Everyday Example: Making a Cup of Tea

Informal (Human):
  1. Boil water.
  2. Put a teabag in a cup.
  3. Pour boiling water into the cup.
  4. Wait a few minutes.
  5. Remove the teabag.
  6. Add milk/sugar if desired.
  7. Drink.
Computer issues:
  • How much water?
  • How hot?
  • What if there are no cups?
  • What if there is no teabag?
  • Infinite loop if water never boils?
  • How much milk/sugar? (or none?)
  • When exactly is it ready to drink?
Programming Mindset When you think like a programmer, you start noticing all the hidden assumptions in everyday instructions. Programming is partly about removing those assumptions.

3.3 Three basic patterns in algorithms

  • Sequence – do one step after another in order.
  • Selection – choose between two or more paths (e.g. “if it is raining, take an umbrella”).
  • Repetition – repeat some steps (e.g. “stir until the sugar dissolves”).

Most programs you write in this module will be built by combining these three patterns. Later, you will see them again as statements in Python: normal lines of code (sequence), if (selection), and while/for (repetition).

3.4 Activity: Write an Everyday Algorithm

Activity

(10–15 minutes, discussion in class or self-reflection)

Pick one of these tasks (or invent your own):

  • Brushing your teeth.
  • Locking up your house when you leave.
  • Making a sandwich.

Write down the steps as if explaining to a very literal robot:

  • Put one action per line.
  • Avoid vague words like “some”, “a bit”, “properly”.
  • Look for decision points: If there is no bread, then…
  • Look for repetition: Stir until no lumps remain.

When you’re done, check:

  • Did you clearly state the starting point (inputs)?
  • Did you clearly state the goal (output)?
  • Could someone else follow your steps without improvising?

4. From Algorithms to Code

A computer cannot understand human languages directly. We need a way to turn our algorithm (idea) into something a computer can execute. This is where programming languages come in.

4.1 Levels of Description

  1. Informal description – “Find the largest of three numbers.”
  2. Algorithm (natural language or pseudocode) – step-by-step instructions that a human can follow.
  3. Code – the algorithm written in a formal programming language like Python.

In this module we will spend a lot of time moving between (2) and (3): from algorithm to code and back again.

4.2 Pseudocode (Not Real Code Yet!)

Pseudocode looks a bit like programming, but it’s written for humans, not machines. There is no single “correct” syntax; the goal is clarity.

Example: find the largest of three numbers a, b, c:

Set largest to a If b is greater than largest: Set largest to b If c is greater than largest: Set largest to c Output largest

Different people might write this slightly differently, and that’s fine. Once the algorithm is clear, we can translate it into Python in a later lecture.

4.3 Sequence, selection, and repetition in code

Remember the three basic patterns of algorithms:

  • Sequence – in code, this is just one statement after another.
  • Selection – implemented with if, elif, else.
  • Repetition – implemented with while and for loops.

In Week 1 and Week 2 we focus more on the ideas. In later weeks you will see exactly how these patterns look in real Python code.

4.4 Activity: Pseudocode for a Simple Problem

(10 minutes)

Activity: Pseudocode

Write pseudocode for one of these tasks:

  • Given three test scores, compute the average and say whether it is a pass or fail.
  • Given a person’s age, decide whether they are a child, teenager, adult, or senior.

Focus on the logic, not on Python syntax. Ask yourself:

  • What are the inputs?
  • What is the output?
  • What decisions (if any) are needed?
  • Is there any repetition?

5. Meet Python

In this module we will use the Python programming language. Python is widely used in web development, data science, artificial intelligence, automation, and more.

  • Its syntax is relatively simple and readable.
  • It runs on many platforms (Windows, macOS, Linux, the web).
  • There is a huge ecosystem of libraries and learning resources.

5.1 How Python runs your code

Python is an interpreted language. That means an interpreter reads your program and executes it line by line. In this course:

  • You will type Python code into an editor in the browser.
  • The browser sends your code to a Python interpreter (Pyodide) running behind the scenes.
  • The interpreter executes your instructions and sends the output back to you.

This setup means you don’t have to install anything to get started.

In this course, you will run Python code directly in your browser using the embedded editor and terminal on this page. In later weeks we will depend on this heavily. For now, here is a single, optional first interaction with Python.
