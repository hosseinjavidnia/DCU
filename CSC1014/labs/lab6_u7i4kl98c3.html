Lab 6 – Practising while Loops

In this lab you will practise writing while loops to repeat actions, count, accumulate values, validate user input, and stop when a special “sentinel” value is entered.

Lab goals
  • Write your own while loops (not just run pre-written examples).
  • Use counters and accumulators to control and summarise repetitions.
  • Use loops for input validation and sentinel-controlled input.
  • Apply while loops to small numeric algorithms (digit sums, counts).
  • Recognise and fix infinite loops and off-by-one errors.
Key terms (used in this lab)
  • Loop: a control structure that repeats a block of code while a condition is true.
  • Iteration: one complete run of the loop body from top to bottom.
  • Accumulator: a variable (such as total) that keeps a running total.
  • Sentinel value: a special value used to signal “stop” (for example, 0 to finish input).
  • Input validation: checking that the user’s input satisfies some condition (for example, is positive) and re-asking if it does not.
  • Infinite loop: a loop that never ends because its condition is always true.
  • Off-by-one error: a mistake where a loop executes one time too many or one time too few.
